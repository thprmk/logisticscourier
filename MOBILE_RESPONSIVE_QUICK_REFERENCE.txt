TAILWIND BREAKPOINTS
================================================================================

Mobile (Default)    : 0px - 640px      (xs, sm screens)
Tablet (sm)         : 640px+           (sm: prefix)
Tablet (md)         : 768px+           (md: prefix)
Desktop (lg)        : 1024px+          (lg: prefix)
Wide Desktop (xl)   : 1280px+          (xl: prefix)


COMMON RESPONSIVE PATTERNS
================================================================================

1. STACKED-TO-GRID LAYOUT
   ✓ Use: Multi-column layouts that stack on mobile
   
   <div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3">
     {/* Mobile: 1 column, Tablet: 2 columns, Desktop: 3 columns */}
   </div>

2. FLEX-RESPONSIVE
   ✓ Use: Flexible layouts with direction changes
   
   <div className="flex flex-col md:flex-row gap-2 md:gap-4">
     {/* Column on mobile, row on tablet+ */}
   </div>

3. HIDDEN-RESPONSIVE (Show/Hide by Screen)
   ✓ Use: Different content for different screens
   
   <span className="hidden md:inline">Full Width Text</span>
   <span className="md:hidden">Compact</span>

4. PADDING-RESPONSIVE
   ✓ Use: Adjust spacing by screen size
   
   <div className="p-4 md:p-6 lg:p-8">
     {/* 16px padding mobile, 24px tablet, 32px desktop */}
   </div>

5. TEXT-RESPONSIVE
   ✓ Use: Scale text for readability
   
   <h1 className="text-2xl md:text-3xl lg:text-4xl">
     Responsive Heading
   </h1>

6. WIDTH-RESPONSIVE
   ✓ Use: Full width on mobile, limited on desktop
   
   <div className="w-full md:w-96 lg:w-[600px]">
     {/* Full width mobile, fixed width desktop */}
   </div>

COMMON MISTAKES TO AVOID
================================================================================

❌ WRONG: Using fixed widths
   <div style={{ width: '800px' }}>
   
✓  RIGHT: Using responsive classes
   <div className="w-full lg:w-[800px]">

❌ WRONG: Text too small on mobile
   <p className="text-xs">Hard to read</p>
   
✓  RIGHT: Scale text responsively
   <p className="text-xs sm:text-sm md:text-base">

❌ WRONG: No gap between mobile buttons
   <div className="flex">
     <Button>1</Button>
     <Button>2</Button>
   </div>
   
✓  RIGHT: Add responsive gaps
   <div className="flex gap-2 md:gap-3">
     <Button>1</Button>
     <Button>2</Button>
   </div>

❌ WRONG: Fixed padding everywhere
   <div className="p-8">Content</div>
   
✓  RIGHT: Responsive padding
   <div className="p-4 md:p-6 lg:p-8">Content</div>

❌ WRONG: Ignoring touch targets
   <button className="p-1">Small</button>
   
✓  RIGHT: Ensure touch-friendly
   <button className="p-3">Large enough</button>

PERFORMANCE TIPS FOR MOBILE
================================================================================

Network
  • Use responsive images
  • Compress images
  • Minimize CSS
  • Cache API responses
  • Lazy load below-fold content

Rendering
  • Use CSS-in-JS sparingly
  • Avoid large DOM trees
  • Defer non-critical CSS
  • Use will-change carefully

User Experience
  • Minimize redirects
  • Preload critical resources
  • Lazy load images
  • Optimize fonts
  • Use service workers
